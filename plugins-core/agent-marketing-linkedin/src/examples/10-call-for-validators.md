Lope ships with 12 built-in AI CLI validators: Claude Code, OpenCode, Gemini CLI, OpenAI Codex, Mistral Vibe, Aider, Ollama, Goose, Open Interpreter, llama.cpp, GitHub Copilot CLI, Amazon Q.

What should be the 13th?

The bar for a built-in adapter is simple: it has to be a real CLI or API you can install today, it has to respond to a prompt, and it has to be different enough from the existing 12 to add diversity to the ensemble. "Yet another OpenAI wrapper" is a no. "Local CLI from a team nobody's integrated yet" is a yes.

A few I'm already watching: Cody from Sourcegraph, Continue.dev's CLI, the Tabby open-source assistant, Phind's CLI if they ship one, Qwen's coder CLI.

Drop a comment with your vote. Or, better, open a PR with the adapter. Two provider types in ~/.lope/config.json cover every backend (subprocess or HTTP), and the README has ready-to-use examples.

Ensemble grows with the community.
